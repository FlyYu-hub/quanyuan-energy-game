# 報告可用段落

## 後端設計說明

本專題以 Flask 作為網站後端伺服器，提供 RESTful API 讓前端取得能源資料、翻牌卡片、問答題庫與排行榜資料。使用者完成翻牌或問答後，前端會透過 POST 請求將遊戲結果回傳後端，後端再將資料儲存於本機 JSON 檔案中，並即時計算玩家學習成果與排行榜。此設計讓原本靜態的前端互動遊戲，升級為具備資料儲存、學習紀錄與成果分析的網站後端應用。

## 資料儲存設計

為避免課程專題產生額外成本，本專題採用本機 JSON 檔案作為資料儲存方式，包含能源資料、問答題庫與玩家遊戲結果。此方式不需付費 API 或雲端資料庫，適合課堂 DEMO 與本機展示；若未來需要擴充，也可將 JSON 儲存替換為 MongoDB，不需大幅改動前端流程。

## 創新與加分點

本專題結合 SDG 7 能源教育與遊戲化學習，透過翻牌配對、問答闖關、知識點數、徽章與排行榜提升使用者參與感。後端不只提供資料，也負責儲存遊戲結果、彙整玩家成果與產生排行榜，使整體系統具備完整前後端資料流。


## 資料刪除與展示安全設計

後端測試面板保留 DEMO 資料建立與 API 狀態檢查，但不提供一鍵清空全站資料，避免展示或多人測試時誤刪所有玩家紀錄。若使用者需要整理自己的紀錄，可在排行榜頁面刪除目前玩家的遊玩資料；後端會依玩家名稱刪除對應紀錄，其他玩家與 DEMO 資料不受影響。

## 線上部署與展示策略

本專題除本機 Flask 展示外，也整理為 Render Web Service 可部署版本。線上版可讓老師與同學透過網址直接開啟作品，不需下載專案；本機版則作為正式上台展示的主要方式，確保後端 API、資料寫入與排行榜功能能穩定展示。由於免費線上服務的本機檔案儲存不適合永久保存，因此本專題將 Render 定位為展示用網址，正式資料流程仍以本機 Flask DEMO 為主。
